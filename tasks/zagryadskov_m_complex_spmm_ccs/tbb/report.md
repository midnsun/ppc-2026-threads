## 1. Контекст

Рассматривается параллельная реализация умножения разреженных матриц в формате **CCS** [1]
с использованием OpenMP [2].

Цель:

- уменьшить время выполнения по сравнению с SEQ
- исследовать масштабируемость алгоритма

---

## 2. Постановка задачи

Необходимо для двух матриц в формате **CCS** вычислить их произведение и затем сохранить его в формате **CCS**.
Произведение двух матриц A = a[i, j], B = b[j, k] (i = 1, m; j = 1, n; k = 1, p; a[i, j] ∈ **C**; b[i, j] ∈ **C**)
равно матрице C со элементами, выражающимися по формуле: C = c[i, k] = Σ(a[i, j] * b[j, k]), j ∈ 1, n.

### Условия корректности

- Проверяется: **A.n = B.m**
- Формат результата: CCS
- Погрешность: **1e-14**
- Сравнение с SEQ

---

## 3. Базовый алгоритм

Используется тот же двухфазный алгоритм, что и в SEQ:

1. Symbolic phase — для каждого столбца правой матрицы определяется структура соответствующего столбца результата:

- Выполняется обход ненулевых элементов и формируется множество индексов строк
- Вычисляется массив **col_ptr** результирующей матрицы

1. Numeric phase — для каждого столбца правой матрицы вычисляется столбец итоговой матрицы:

- Для каждого столбца используются локальные для потоков вектора **acc** аккумулирования результата и **marker**
- Численный результат записывается в заранее выделенные векторы результирующей матрицы

Параллелизм вводится на уровне обработки столбцов матрицы B. При этом столбцы делятся на диапазоны
`blocked_range`, рабочие массивы `acc` и `marker` хранятся в локальном для потоков контейнере

---

## 4. Схема распараллеливания

### Область параллелизма

Параллелизуется цикл по столбцам. Выбранный примитив - `oneapi::tbb::parallel_for`. При этом
экспериментально подобран `grain_size` равный 64. Таким образом, обеспечивается выдача не 1 столбца
на задачу, а сразу нескольких (в данном случае 64х). Приведенный ниже код иллюстрирует разбиение
умножения на задачи по потокам:

```cpp
tbb::parallel_for(tbb::blocked_range<int>(0, n, 64), [&](const tbb::blocked_range<int> &r) {
  auto &marker = tls_marker.local();
  auto &acc = tls_acc.local();
  std::vector<int> rows;
  for (int j = r.begin(); j < r.end(); ++j) {
    SpMMKernel(a, b, c, zero, rows, acc, marker, j);
  }
});
```

`partitioner` явно не задается, используется поведение oneTBB по умолчанию.

## 5. Детали реализации

Файлы:

- `seq/include/ops_tbb.hpp`
- `seq/src/ops_tbb.cpp`

Pipeline реализации:

- **ValidationImpl**
  - проверка условия `A.n == B.m`

- **PreProcessingImpl**
  - входные данные уже готовы

- **RunImpl**
  - Вызов функции SpMM
  - Функция SpMM вызывает фазы Symbolic и Numeric
  - Каждая из фаз содержит внутри себя параллелизм

- **PostProcessingImpl**
  - результат уже сформирован

### Особенности

- Используется двухфазный подход, засчёт которого обеспечивается отсутствие гонок данных
- `tbb::enumerable_thread_specific<std::vector<std::complex<double>>>` обеспечивает каждому потоку
собственные локальные векторы данных
- Благодаря двухфазному подходу практически весь код выполняется параллельно, отсутствуют последовательные части

  - случайные разреженные матрицы

Сравнение выполняется с точностью **1e-14**.

Параметры эксперимента:

- Тип: ленточные разреженные матрицы  
- Ширина ленты: 100

| workers | time (s) | speedup | efficiency |
| -------- | --------- | -------- | ------------ |
| 1 | 1.043 | 0.87 | 0.87 |
| 2 | 0.604 | 1.58 | 0.79 |
| 3 | 0.390 | 2.44 | 0.81 |
| 6 | 0.257 | 3.70 | 0.62 |

Наблюдения:

- Почти рост эффективости до 3 потоков
- Начиная с 6 потоков эффективность падает  
- Причины:
  - при увеличении количества потоков возрастает нагрузка на балансировщик
  - ограничение общей шины данных

## Приложение

Фрагмент показывает базовая функция Numeric, на которой иллюстрируется принцип распределения
данных по потокам с помощью TBB. Исспользуется thread-local контейнер `tbb::enumerable_thread_specific`,
а также `tbb::parallel_for` для распределения задач.

```cpp
void ZagryadskovMComplexSpMMCCSTBB::SpMMNumeric(const CCS &a, const CCS &b, CCS &c, const std::complex<double> &zero) {
  const int m = a.m;
  const int n = b.n;

  tbb::enumerable_thread_specific<std::vector<int>> tls_marker([&]() { return std::vector<int>(m, -1); });

  tbb::enumerable_thread_specific<std::vector<std::complex<double>>> tls_acc(
      [&]() { return std::vector<std::complex<double>>(m, zero); });

  tbb::parallel_for(tbb::blocked_range<int>(0, n, 64), [&](const tbb::blocked_range<int> &r) {
    auto &marker = tls_marker.local();
    auto &acc = tls_acc.local();

    std::vector<int> rows;

    for (int j = r.begin(); j < r.end(); ++j) {
      SpMMKernel(a, b, c, zero, rows, acc, marker, j);
    }
  });
}
```
