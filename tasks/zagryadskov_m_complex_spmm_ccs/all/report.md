## 1. Контекст

Рассматривается гибридная параллельная реализация умножения разреженных матриц в формате **CCS** [1],  
использующая:

- распределённую память (MPI [2])
- общую память (OpenMP [3])

Цель:

- создание реализации, эффективно использующей технологии `OpenMP` и `MPI` совместно
- масштабирование алгоритма на уровне процессов и потоков

---

## 2. Постановка задачи

Необходимо для двух матриц в формате **CCS** вычислить их произведение и затем сохранить его в формате **CCS**.
Произведение двух матриц A = a[i, j], B = b[j, k] (i = 1, m; j = 1, n; k = 1, p; a[i, j] ∈ **C**; b[i, j] ∈ **C**)
равно матрице C со элементами, выражающимися по формуле: C = c[i, k] = Σ(a[i, j] * b[j, k]), j ∈ 1, n.

### Условия корректности

- Проверка: **A.n = B.m**
- Формат результата: CCS
- Точность: **1e-14**
- Сравнение с SEQ

---

## 3. Алгоритм (MPI уровень)

### Общая схема

На уровне процессов алгоритм состоит из нескольких этапов:

1. **Broadcast A**
   - матрица **A** рассылается всем процессам (`MPI_Bcast`)

2. **Разбиение B**
   - столбцы **B** распределяются по процессам на корневом процессе
   - из этих столбцов формируются подматрицы `local_B`
   - `local_B` рассылаются соответствующим процессам

3. **Локальное умножение**
   - каждый процесс вычисляет **A * local_B = local_C**
   - для этого используется распараллеливание на уровне потоков

4. **Сбор результата**
   - `local_C` собираются на корневом процессе (`MPI_Gather/Gatherv`)
   - на корневом процессе формируется итоговая матрица **C**

5. **Broadcast результата**
   - матрица **C** рассылается всем процессам для обеспечения успешного прохождения
тестов всеми процессами

---

### Разбиение данных

- декомпозиция по столбцам:
  -`j_start = rank / size * n`;
  -`j_start = (rank + 1) / size * n`,

где `rank` - номер процесса, `size` - количество процессов, `n` - размерность матрицы **B** по столбцам.

---

### Узкие места

- корневой процесс выполняет:
  - рассылку матрицы **A**
  - распределение и рассылку столбцов матрицы **B**
  - сборку массива `col_ptr` матрицы **C**, конкатенацию остальных массивов
- коммуникации:
  - передача CCS-структур
- возможный дисбаланс при неравномерной разреженности

---

## 4. Алгоритм (OpenMP уровень)

Внутри каждого процесса используется алгоритм из OMP:

### Двухфазная схема

1. Фаза **Symbolic**
   - построение масстива `col_ptr` матрицы **local_C**
   - выделение памяти для **local_C**

2. Фаза **Numeric**
   - вычисление значений матрицы **local_C**
   - запись результата в заранее выделенную память
   - Для каждого столбца используются локальные для потоков вектора **acc** аккумулирования результата и **marker**

Параллелизм вводится на уровне обработки столбцов матрицы **local_B**.

---

## 5. Схема распараллеливания

### MPI уровень

- процессы независимы
- обмен через:
  - `MPI_Bcast` (матрица **A**)
  - `MPI_Send / Recv` (матрица **B**)
  - `MPI_Gather / Gatherv` (сборка матрицы **C**)

---

### OpenMP уровень

#### Область параллелизма

- по столбцам `local_B`
- каждый поток обрабатывает свой диапазон `[ j_start(thread), j_end(thread) )`

#### Разделение данных

- **shared**:
  - `a`, `local_b`, `local_c`

- **private**:
  - `marker`, `acc`, `rows` (локальные внутри функций)

#### Синхронизация

- отсутствует
- каждый поток пишет в свою область подматрицы **local_C**

#### Барьеры

- неявный барьер в конце `omp parallel`

#### Распределение работы

Также используется ручное разбиение диапазона:
`j_start(tid) = (tid * local_B.n) / num_threads`;
`j_end(tid) = ((tid + 1) * local_B.n) / num_threads`,
где tid - номер потока, num_threads - общее число потоков

## 6. Детали реализации

Pipeline:

- **ValidationImpl**
  - проверка размеров (на нулевом процессе)

- **PreProcessingImpl**
  - данные уже готовы

- **RunImpl**
  - MPI:
    - `BcastCCS`
    - `ScatterB`
    - локальный `SpMM`
    - `GatherC`
  - OMP (внутри SpMM):
    -параллельный вызов:
    - `SpMMSymbolic`
    - `SpMMNumeric`

- **PostProcessingImpl**
  - рассылка результата

### Ключевая идея

- MPI → распределяет данные и запускает вычисление SpMM локально для подматриц
- OpenMP → ускоряет локальные вычисления

`Bcast`, `Send`, `Recv` и `Gather` передают в том числе комплексные числа, хранящиеся в типе данных
`std::complex<double>`. Для передачи используется обратная совместимость типа `std::complex` с типом данных MPI `MPI_C_DOUBLE_COMPLEX`.

### Наблюдения

- выигрыш от MPI сопоставим с OpenMP для малого размера процессов

- наблюдается рост ускорения с увеличением числа `workers`, что подтверждает хорошую производительность
реализации

Причины падения эффективности:

- коммуникации MPI  
- последовательные участки на корневом процессе  
- ограничение памяти: слишком большая нагрузка на шину данных при большом числе `workers`

## Приложение

В приведённом ниже листинге функции `RunImpl` демонстрируется общий подход к распределению
данных на уровне процессов и выполнение последующего произведения подматриц на каждом процессе

```cpp
bool ZagryadskovMComplexSpMMCCSALL::RunImpl() {
  int world_rank = 0;
  int world_size = 0;
  MPI_Comm_rank(MPI_COMM_WORLD, &world_rank);
  MPI_Comm_size(MPI_COMM_WORLD, &world_size);
  CCS &a = std::get<0>(GetInput());
  CCS &b = std::get<1>(GetInput());
  CCS &c = GetOutput();

  CCS local_b;
  CCS local_c;
  std::vector<int> col_starts;
  if (world_rank == 0) {
    col_starts.resize(world_size + 1);
    for (int proc = 0; proc <= world_size; ++proc) {
      col_starts[proc] = (proc * b.n) / world_size;
    }
  }

  BcastCCS(a, world_rank);
  ScatterB(b, local_b, col_starts, world_rank, world_size);

  ZagryadskovMComplexSpMMCCSALL::SpMM(a, local_b, local_c);

  GatherC(c, local_c, world_rank, world_size);

  return true;
}
```

На примере функции отправки матрицы `SendCCS` приложенный ниже листинг демонстрирует коммуникацию
MPI-процессов при передачи матриц в `CCS` - формате:

```cpp
void ZagryadskovMComplexSpMMCCSALL::SendCCS(const CCS &m, int dest) {
  MPI_Send(&m.m, 1, MPI_INT, dest, 0, MPI_COMM_WORLD);
  MPI_Send(&m.n, 1, MPI_INT, dest, 0, MPI_COMM_WORLD);
  int nz = static_cast<int>(m.values.size());
  MPI_Send(&nz, 1, MPI_INT, dest, 0, MPI_COMM_WORLD);

  MPI_Send(m.col_ptr.data(), m.n + 1, MPI_INT, dest, 0, MPI_COMM_WORLD);
  MPI_Send(m.row_ind.data(), nz, MPI_INT, dest, 0, MPI_COMM_WORLD);
  MPI_Send(m.values.data(), nz, MPI_C_DOUBLE_COMPLEX, dest, 0, MPI_COMM_WORLD);
}
```
